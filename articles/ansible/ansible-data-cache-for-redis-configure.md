# <a name="tutorial-configure-caches-in-azure-cache-for-redis-using-ansible"></a>Tutorial: Configurar os caches no Cache Redis do Azure usando o Ansible

[!INCLUDE [ansible-28-note.md](../../includes/ansible-28-note.md)]

O [Cache Redis do Azure](/azure/azure-cache-for-redis/) é um serviço compatível de código-fonte aberto que permite criar aplicativos responsivos fornecendo acesso rápido aos dados. 

[!INCLUDE [ansible-tutorial-goals.md](../../includes/ansible-tutorial-goals.md)]

> [!div class="checklist"]
>
> * Criar um cache
> * Dimensionar um cache
> * Reinicializar um cache
> * Adicionar uma regra de firewall a um cache
> * Excluir um cache

## <a name="create-a-cache"></a>Criar um cache

Crie um Cache Redis do Azure dentro de um novo grupo de recursos.

```yml
  - name: Create resource group
    azure_rm_resourcegroup:
      name: "{{ resource_group }}"
      location: "{{ location }}"

  - name: Create Azure Cache for Redis
    azure_rm_rediscache:
      resource_group: "{{ resource_group }}"
      name: "{{ redis_name }}"
      sku:
        name: basic
        size: C1 
```

O provisionamento de um cache pode levar vários minutos. O código a seguir informa o Ansible para aguardar a conclusão da operação:

```yml
  - name: Wait for Redis provisioning to complete
    azure_rm_rediscache_facts:
      resource_group: "{{ resource_group }}"
      name: "{{ redis_name }}"
    register: facts
    until: "{{ facts.rediscaches[0]['provisioning_state'] == 'Succeeded' }}"
    retries: 100
    delay: 60
```

Durante o longo processo de provisionamento, várias mensagens de "erro" serão exibidas. Essas mensagens podem ser ignoradas com segurança. A mensagem importante é a última mensagem. No exemplo a seguir, há muitas mensagens de erro até a mensagem final ("ok").

## <a name="scale-the-cache"></a>Dimensionar o cache

O Cache Redis do Azure tem várias ofertas de cache dependendo das necessidades do aplicativo. Essas opções de cache oferecem flexibilidade na escolha do tamanho e recursos do cache. Se seus requisitos de aplicativo mudarem após a criação do cache, você poderá dimensionar o cache conforme necessário. Para saber mais sobre o dimensionamento, confira [Como dimensionar o Cache Redis do Azure](/azure/azure-cache-for-redis/cache-how-to-scale).

O código de exemplo a seguir dimensiona o cache para **Standard**:

```yml
- name: Scale up Azure Cache for Redis
    azure_rm_rediscache:
      resource_group: "{{ resource_group }}"
      name: "{{ redis_name }}"
      sku:
        name: standard
        size: C1
```

O dimensionamento de um cache pode levar vários minutos. O código a seguir informa o Ansible para aguardar a conclusão da operação:

```yml
  - name: Wait for Redis scaling up to complete
    azure_rm_rediscache_facts:
      resource_group: "{{ resource_group }}"
      name: "{{ redis_name }}"
    register: facts
    until: "{{ facts.rediscaches[0]['provisioning_state'] == 'Succeeded' }}"
    retries: 100
    delay: 60
```

Semelhante à tarefa de provisionamento do Cache Redis do Azure, a saída como a mensagem a seguir é normal:

```Ouput
**FAILED - RETRYING: Get facts (100 retries left)** is normal.
```

## <a name="reboot-the-cache"></a>Reinicializar o cache

O código a seguir reinicializa o cache criado nas seções anteriores.

```yml
  - name: Reboot Azure Cache for Redis
    azure_rm_rediscache:
      resource_group: "{{ resource_group }}"
      name: "{{ redis_name }}"
      reboot:
        reboot_type: all
```

### <a name="add-firewall-rule"></a>Adicionar regra de firewall

O código a seguir adiciona uma regra de firewall ao cache:

```yml
  - name: Add Firewall rule
    azure_rm_rediscachefirewallrule:
      resource_group: "{{ resource_group }}"
      cache_name: "{{ redis_name }}"
      name: rule1
      start_ip_address: 168.1.1.1
      end_ip_address: 168.1.1.4
```

## <a name="delete-the-cache"></a>Excluir o cache

O código a seguir exclui o cache:

```yml
  - name: Delete Azure Cache for Redis
    azure_rm_rediscache:
      resource_group: "{{ resource_group }}"
      name: "{{ redis_name }}"
      state: absent
```

      resource_group: "{{ resource_group }}"

      resource_group: "{{ resource_group }}"
      name: "{{ redis_name }}"

        name: standard
        size: C1

  - name: Wait for Redis scaling up to complete
    azure_rm_rediscache_facts:
      resource_group: "{{ resource_group }}"
      name: "{{ redis_name }}"
    register: facts
    until: "{{ facts.rediscaches[0]['provisioning_state'] == 'Succeeded' }}"
    retries: 100
    delay: 60

  - name: Reboot Azure Cache for Redis
    azure_rm_rediscache:
      resource_group: "{{ resource_group }}"
      name: "{{ redis_name }}"
      reboot:
        reboot_type: all

  - name: Add Firewall rule
    azure_rm_rediscachefirewallrule:
      resource_group: "{{ resource_group }}"
      cache_name: "{{ redis_name }}"
      name: rule1
      start_ip_address: 168.1.1.1
      end_ip_address: 168.1.1.4

  - name: Delete Azure Cache for Redis
    azure_rm_rediscache:

## <a name="clean-up-resources"></a>Limpar recursos

Quando não forem mais necessários, exclua os recursos criados neste artigo. 

Salve o seguinte código como `cleanup.yml`:

```yml
- hosts: localhost
  vars:
    resource_group: "{{ resource_group_name }}"
  tasks:
    - name: Delete a resource group
      azure_rm_resourcegroup:
        name: "{{ resource_group }}"
        state: absent
```

Na seção `vars`, substitua o espaço reservado `{{ resource_group_name }}` pelo nome do grupo de recursos.

Execute o guia estratégico usando o comando `ansible-playbook`:

```bash
ansible-playbook cleanup.yml
```
